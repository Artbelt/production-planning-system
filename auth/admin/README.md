# 👥 Административная панель

Полнофункциональная система управления пользователями и правами доступа для единой системы авторизации.

## 🚀 Доступ к панели

**URL:** `http://localhost/auth/admin/`

**Требования для доступа:**
- Авторизованный пользователь
- Роль "Директор" в любом из цехов

**Логин администратора по умолчанию:**
- Телефон: `+380995527932`
- Пароль: `7932`

## 📋 Структура панели

### 🏠 Главная страница (`index.php`)
- **Общая статистика** системы
- **Последние действия** пользователей
- **Активные сессии**
- Быстрый доступ к основным разделам

### 👥 Управление пользователями (`users.php`)
**Функции:**
- ✅ Просмотр всех пользователей с их ролями
- ✅ Создание новых пользователей
- ✅ Активация/деактивация пользователей
- ✅ Переход к редактированию пользователя

**Создание пользователя:**
1. Нажмите "➕ Добавить пользователя"
2. Заполните форму:
   - Номер телефона (с форматированием)
   - ФИО
   - Пароль
3. Пользователь создается активным и верифицированным

### ✏️ Редактирование пользователя (`user-edit.php`)
**Функции:**
- ✅ Изменение основных данных (телефон, ФИО, email)
- ✅ Смена пароля
- ✅ **Назначение ролей по цехам** (основная функция)
- ✅ Просмотр истории действий пользователя

**Назначение прав доступа:**
- Для каждого цеха можно назначить роль:
  - **Рабочий** - базовый доступ
  - **Менеджер** - расширенные права
  - **Мастер** - управление цехом
  - **Директор** - полные права + админ панель
- Один пользователь может иметь разные роли в разных цехах
- Можно убрать доступ к цеху, выбрав "Нет доступа"

### 🔐 Управление ролями (`roles.php`)
**Функции:**
- ✅ Просмотр всех ролей системы
- ✅ Создание пользовательских ролей
- ✅ Активация/деактивация ролей
- ✅ Статистика распределения ролей по цехам

**Системные роли** (нельзя деактивировать):
- `worker` - Рабочий
- `manager` - Менеджер  
- `supervisor` - Мастер
- `director` - Директор

**Создание пользовательской роли:**
1. Нажмите "➕ Добавить роль"
2. Укажите системное имя (только буквы и подчеркивания)
3. Укажите отображаемое имя
4. Добавьте описание

### 🔄 Управление сессиями (`sessions.php`)
**Функции:**
- ✅ Просмотр всех активных сессий в реальном времени
- ✅ Завершение отдельных сессий
- ✅ Завершение всех сессий пользователя
- ✅ Очистка истекших сессий
- ✅ Статистика по сессиям и IP адресам
- ✅ Автообновление каждые 30 секунд

**Информация о сессиях:**
- ID сессии и время создания
- Пользователь (ФИО и телефон)
- Текущий цех работы
- IP адрес и время последней активности
- Время до истечения сессии
- Статус активности (активен/неактивен/истекает)

**Массовые действия:**
- Завершить все сессии пользователя
- Очистить все истекшие сессии
- Обновить данные в реальном времени

### 📊 Посещения страниц (`page-stats.php`)
**URL:** `.../auth/admin/page-stats.php`

**Функции:**
- Топ страниц по числу открытий (путь `SCRIPT_NAME`)
- Активность по пользователям (ФИО, телефон)
- Лента последних просмотров с IP и временем
- Период: 1 / 7 / 30 / 90 дней

**Источник данных:** записи `auth_logs` с действием `page_view` создаются при каждом успешном вызове `AuthManager::checkSession()` (кроме скриптов в `/auth/api/`). Страницы, которые не вызывают проверку сессии, в статистике не отображаются.

### 📋 Логи системы (`logs.php`)
**Функции:**
- ✅ Просмотр всех действий в системе
- ✅ Фильтрация по:
  - Типу действия (login, logout, failed_login, etc.)
  - Пользователю
  - Диапазону дат
  - IP адресу
- ✅ Пагинация (50 записей на страницу)
- ✅ Статистика активности

**Типы действий в логах:**
- `login` - Успешный вход
- `logout` - Выход из системы
- `failed_login` - Неудачная попытка входа
- `department_switch` - Смена цеха
- `page_view` - Открытие страницы (путь в поле `details`, JSON)
- `session_expired` - Истечение сессии
- `session_terminated_by_admin` - Сессия завершена администратором
- `all_sessions_terminated_by_admin` - Все сессии пользователя завершены
- `expired_sessions_cleanup` - Очистка истекших сессий

## 🔒 Система безопасности

### Контроль доступа
- Доступ только для пользователей с ролью "Директор"
- Проверка прав на каждой странице
- Автоматическое перенаправление при отсутствии прав

### Логирование
- Все действия пользователей записываются в `auth_logs`
- Сохраняется IP адрес, User Agent, детали действия
- Логи доступны для анализа в админ панели

### Защита от CSRF
- Все формы защищены CSRF токенами
- Проверка токенов на серверной стороне

## 🔧 Техническая информация

### Файловая структура
```
auth/admin/
├── index.php          # Главная страница
├── page-stats.php     # Статистика посещений страниц
├── users.php          # Управление пользователями
├── user-edit.php      # Редактирование пользователя
├── roles.php          # Управление ролями
├── sessions.php       # Управление сессиями
├── logs.php           # Просмотр логов
└── README.md          # Документация

auth/api/
└── sessions.php       # API для управления сессиями
```

### Зависимости
- PHP 7.4+
- MySQL/MariaDB
- Система авторизации `auth/`
- CSS фреймворк из `auth/assets/css/auth.css`

### База данных
Использует таблицы из схемы `auth`:
- `auth_users` - пользователи
- `auth_roles` - роли
- `auth_user_departments` - назначения ролей по цехам
- `auth_sessions` - активные сессии
- `auth_logs` - логи действий
